[설계안] Harrier-OSS-v1 기반 AI IT Helpdesk 시스템1. 개요 (Overview)본 문서는 마이크로소프트의 최신 오픈소스 임베딩 모델인 Harrier-OSS-v1을 활용하여, 기업 내 IT 기술 지원 업무를 자동화하는 AI IT Helpdesk 시스템의 아키텍처와 구현 방안을 정의합니다.2. 시스템 아키텍처 (System Architecture)2.1 데이터 흐름 (Data Flow)User Input: 사용자가 웹 인터페이스를 통해 IT 관련 질문 입력Embedding (Harrier): 유저의 질문을 Harrier-OSS-v1 모델을 통해 고차원 벡터로 변환Vector Search (ChromaDB): 변환된 벡터를 사용하여 ChromaDB 내의 IT 가이드/FAQ 인덱스에서 유사 문서 검색RAG Prompting: 검색된 컨텍스트(Context)와 유저 질문을 결합하여 LLM(Gemini 등) 전달Response Generation: LLM이 최종 해결 방법을 생성하여 사용자에게 답변2.2 하이브리드 구성 제안구분적용 모델역할Indexing/RetrievalHarrier-0.6B방대한 IT 매뉴얼의 고속 검색 및 벡터 데이터베이스 구축Re-ranking (선택)Harrier-27B검색된 상위 결과 중 질문 의도와 가장 일치하는 문서 정밀 재추출Final AnswerLLM (Gemini 3)최종 사용자 응답 생성 및 문제 해결 단계 가이드3. 핵심 적용 기술 및 장점 (Key Features)32K 컨텍스트 지원: 수십 페이지 분량의 복잡한 네트워크 장애 대응 매뉴얼이나 보안 규정집을 데이터 손실 없이 한 번에 처리 가능합니다.다국어 성능(SOTA): 한국어 이해도가 매우 뛰어나 "프린터가 안 잡혀요", "VPN 접속 오류" 등 현장 중심의 표현을 정확히 인식합니다.비용 및 보안 최적화: 외부 임베딩 API 호출 없이 사내 서버(AMS) 또는 로컬 환경(Jetson/Mac)에서 독립적으로 운영되어 데이터 보안을 강화합니다.4. 구현 가이드라인 (Implementation)4.1 지시어(Instruction) 활용Harrier 모델은 지시어 튜닝 모델이므로, 검색 시 다음 형식을 권장합니다.Query Template:“주어진 IT 헬프데스크 질문에 대해 가장 관련 있는 해결 방법을 찾으세요: {User_Question}”4.2 시스템 구성 요소 예시구성 요소기술 스택FrontendReact / Next.jsBackend APIFastAPI (Python)Embedding ModelMicrosoft Harrier-OSS-v1-0.6BVector DBChromaDBLLMGoogle Gemini 3 (Flash/Pro)5. 기대 효과 (Business Impact)응답 효율화: 단순 반복적인 IT 문의(비밀번호 초기화, 소프트웨어 설치 등)를 80% 이상 자동화데이터 내재화: 기업 고유의 기술 자산을 외부 유출 없이 AI 지식 자산으로 전환확장성: AMS의 하이브리드 에이전트 OS 내 기업용 모듈로 즉시 이식 가능6. 향후 과제Jetson Thor 환경에서의 Harrier-0.6B 추론 속도 최적화 테스트ChromaDB와 Harrier 모델 간의 매끄러운 랭킹 파이프라인 구축 및 검증
