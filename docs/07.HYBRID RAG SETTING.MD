# 07. Hybrid RAG Setting

## Current Behavior

The project supports a hybrid KB strategy:

1. Preferred path: ChromaDB vector retrieval
2. Fallback path: database text matching on `kb_documents`

This means the application can continue working even when ChromaDB is not connected.

## Relevant Settings

- `CHROMA_HOST`
- `CHROMA_PORT`
- `CHROMA_COLLECTION`
- `rag_enabled` in `system_settings`
- `rag_top_k` in `system_settings`

## Runtime Rules

- If `rag_enabled=false`, chat skips KB retrieval
- If `rag_enabled=true` and ChromaDB is available, vector search is used
- If ChromaDB is unavailable, fallback search uses database text matching

## Current Limitation

Fallback search is functional but much less accurate than vector retrieval.
