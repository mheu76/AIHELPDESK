# 05. UI 화면 설계

> Frontend: Next.js 14 (App Router) + TypeScript + Tailwind CSS + shadcn/ui

---

## 1. 라우팅 구조

```
/                       → /chat (리다이렉트)
/login                  → 로그인 화면
/chat                   → 채팅 메인 (임직원)
/chat/[sessionId]       → 채팅 세션 상세
/tickets                → 티켓 목록 (임직원: 본인 / IT담당자: 전체)
/tickets/[ticketId]     → 티켓 상세
/admin                  → 관리자 대시보드
/admin/kb               → KB 문서 관리
/admin/users            → 사용자 관리
/admin/settings         → 시스템 설정
```

## 3. 화면별 상세 설계

### 3-1. 로그인 (/login)

```
┌─────────────────────────────┐
│        IT Helpdesk          │
│          [로고]              │
│                             │
│  이메일 [________________]  │
│  비밀번호 [______________]  │
│                             │
│      [로그인]               │
└─────────────────────────────┘
```

**컴포넌트:** `LoginForm`
**동작:**
- 로그인 성공 → `/chat` 이동
- 실패 → 인라인 에러 메시지

---

### 3-2. 채팅 메인 (/chat)

```
┌──────────┬──────────────────────────────────────┐
│ [+ 새채팅]│                                      │
│──────────│        채팅을 시작해보세요             │
│ 오늘      │                                      │
│ 비밀번호  │   자주 묻는 질문:                     │
│ 문의      │   [비밀번호 초기화] [VPN 연결 오류]   │
│──────────│   [소프트웨어 설치] [권한 신청]        │
│ 어제      │                                      │
│ VPN 오류  │                                      │
└──────────┴──────────────────────────────────────┘
```

**컴포넌트:** `ChatSidebar`, `ChatWelcome`
**동작:**
- "새 채팅" 클릭 → 새 세션 생성 → `/chat/[sessionId]`
- 자주 묻는 질문 클릭 → 해당 질문으로 채팅 시작

---

### 3-3. 채팅 상세 (/chat/[sessionId])

```
┌──────────┬──────────────────────────────────────┐
│ [+ 새채팅]│  비밀번호 초기화 문의          [티켓] │
│──────────│──────────────────────────────────────│
│ 채팅목록  │                                      │
│          │  👤 비밀번호를 초기화하고 싶어요        │
│          │                              오후 2:30 │
│          │                                      │
│          │  🤖 비밀번호 초기화 방법을 안내드립니다 │
│          │     1. [사내 포털] 접속               │
│          │     2. 우측 상단 프로필 클릭           │
│          │     3. 보안 설정 → 비밀번호 변경       │
│          │                                      │
│          │  ┌─────────────────────────────────┐ │
│          │  │ 도움이 되셨나요?  [해결됨] [티켓] │ │
│          │  └─────────────────────────────────┘ │
│          │──────────────────────────────────────│
│          │  [메시지 입력...              ] [전송] │
└──────────┴──────────────────────────────────────┘
```

**컴포넌트:** `ChatMessages`, `ChatInput`, `ChatFeedback`
**동작:**
- AI 응답: SSE Streaming으로 실시간 표시
- "해결됨": 세션 resolved 처리
- "티켓": 티켓 생성 모달 → 확인 후 생성

---

### 3-4. 티켓 목록 (/tickets)

```
┌─────────────────────────────────────────────────┐
│  내 티켓 목록              [상태▼] [유형▼] [검색] │
├──────┬──────────────┬──────┬──────┬─────────────┤
│ 번호  │ 제목         │ 유형  │ 상태  │ 생성일      │
├──────┼──────────────┼──────┼──────┼─────────────┤
│T-001 │ 비밀번호 초기화│ 계정  │처리중 │ 2025-01-01 │
│T-002 │ VPN 연결 오류 │ 네트워크│완료  │ 2024-12-30 │
└──────┴──────────────┴──────┴──────┴─────────────┘
```

**상태 배지 색상:**
- 접수: 회색
- 처리중: 파랑
- 완료: 초록
- 보류: 노랑
