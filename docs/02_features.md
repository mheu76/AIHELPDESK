# 02. 기능 스펙

---

## 1. 사용자 유형

| 유형 | 설명 | 권한 |
|---|---|---|
| **임직원** | 일반 사용자, IT 문의 제출 | 채팅, 본인 티켓 조회 |
| **IT 담당자** | 티켓 처리 담당 | 전체 티켓 관리, KB 관리 |
| **관리자** | 시스템 전체 관리 | 전체 권한 + 통계 + LLM 설정 |

---

## 2. 기능 목록

### Sprint 1 — 채팅 기본 기능

#### F-001. 로그인
- **설명:** JWT 기반 로그인 (추후 SSO 교체 가능 구조)
- **입력:** 사번(이메일), 비밀번호
- **출력:** Access Token, Refresh Token
- **예외:** 잘못된 자격증명 → 401 에러

#### F-002. AI 채팅
- **설명:** 임직원이 IT 문의를 채팅으로 입력하면 AI가 KB 기반으로 1차 응답
- **입력:** 질문 텍스트
- **출력:** AI 답변 텍스트 (Streaming 지원)
- **RAG:** ChromaDB에서 관련 문서 Top-3 검색 후 컨텍스트 주입
- **세션:** 대화 맥락 유지 (최근 10턴)
- **예외 처리:**
  - KB에 없는 내용 → "티켓으로 전달하시겠습니까?" 안내
  - LLM API 오류 → 담당자 직접 연락 안내

#### F-003. 대화 이력 저장
- **설명:** 모든 채팅 대화를 DB에 저장
- **저장 항목:** 세션ID, 유저ID, role(user/assistant), 메시지, 타임스탬프

---

### Sprint 2 — RAG / KB 관리

#### F-004. KB 문서 업로드 (IT 담당자)
- **설명:** IT 매뉴얼, FAQ, 정책 문서를 업로드하여 ChromaDB에 벡터화 저장
- **지원 형식:** PDF, DOCX, TXT, MD
- **처리 과정:** 업로드 → 텍스트 추출 → 청크 분할 → 임베딩 → ChromaDB 저장
- **UI:** 파일 드래그앤드롭 + 업로드 진행률 표시

#### F-005. KB 문서 목록·삭제 (IT 담당자)
- **설명:** 등록된 KB 문서 목록 조회 및 삭제
- **표시 항목:** 문서명, 업로드일, 청크 수, 업로드자

---

### Sprint 3 — 티켓 시스템

#### F-006. 티켓 자동 생성
- **설명:** AI가 해결 못한 문의를 티켓으로 전환
- **트리거:**
  - 임직원이 "티켓 생성" 버튼 클릭
  - AI가 3회 연속 해결 불가 판단 시 자동 안내
- **자동 처리:**
  - 대화 내용 AI 요약 → 티켓 제목·내용 자동 생성
  - 문의 유형 자동 분류 (계정/장비/네트워크/시스템/보안)
- **출력:** 티켓 번호 발급

#### F-007. 티켓 조회 (임직원)
- **설명:** 본인이 생성한 티켓 목록 및 상태 확인
- **상태:** 접수 → 처리중 → 완료 → 보류
- **표시 항목:** 티켓번호, 제목, 유형, 상태, 생성일, 처리 담당자

#### F-008. 티켓 처리 (IT 담당자)
- **설명:** 배정된 티켓 확인 및 처리
- **기능:**
  - 티켓 목록 조회 (필터: 상태, 유형, 날짜)
  - 티켓 상세 조회 (대화 이력 포함)
  - 상태 변경
  - 처리 내용 작성
  - 담당자 재배정

#### F-009. 티켓 알림
- **설명:** 티켓 이벤트 발생 시 관련자에게 알림
- **알림 시점:**
  - 신규 티켓 생성 → IT 담당자
  - 티켓 상태 변경 → 임직원
- **방식:** 내부 웹 알림 (Phase 1) / 이메일 (Phase 2)

---

### Sprint 4 — 관리자 기능

#### F-010. 통계 대시보드 (관리자)
- **설명:** 시스템 운영 현황 모니터링
- **표시 항목:**
  - 일별·월별 문의 건수
  - AI 해결률 (티켓 전환율)
  - 평균 응답 시간
  - 문의 유형별 분포
  - 미처리 티켓 현황

#### F-011. LLM Provider 설정 (관리자)
- **설명:** LLM Provider를 UI에서 전환
- **기능:** 현재 Provider 확인, Provider 변경 (claude/openai), API 키 설정
- **적용:** 변경 즉시 반영 (서버 재시작 불필요)

#### F-012. 사용자 관리 (관리자)
- **설명:** 임직원 계정 및 권한 관리
- **기능:** 계정 생성·수정·비활성화, 역할 변경 (임직원/IT담당자/관리자)

## 3. 비기능 요구사항

| 항목 | 요구사항 |
|---|---|
| **응답 속도** | AI 첫 토큰 응답 3초 이내 (Streaming) |
| **동시 접속** | 50명 동시 접속 처리 |
| **가용성** | 업무 시간 내 99% 이상 |
| **데이터 보존** | 대화 이력 1년, 티켓 3년 |
| **브라우저 지원** | Chrome, Edge 최신 버전 |
